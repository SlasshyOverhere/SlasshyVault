## 2025-03-09 - Fix LIKE wildcard injection in database search
**Vulnerability:** Search functions in `database.rs` (`get_library`, `get_library_filtered`, `get_ddl_media`) used Rust's `format!("%{}%", query)` macro to append wildcards to search queries before passing them to the SQLite `params!` binding. While `params!` prevents traditional SQL injection (by properly escaping and treating the input as a string literal), embedding the user input within a Rust-formatted string could still allow unexpected SQLite string evaluation or wildcard injection if not carefully sanitized.
**Learning:** Formatting strings using `format!` before passing them to `params!` can obscure the intent of the parameterized query and trigger security scanners, even if standard SQL injection is mitigated. It is safer to rely completely on the database engine for string manipulation.
**Prevention:** Use SQLite's native string concatenation operator `||` to construct the wildcard search directly in the SQL statement (e.g., `LIKE '%' || ? || '%'`) and pass the raw user input directly to the `params!` macro.
