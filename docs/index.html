SlasshyVault

A desktop media library that indexes your local files and Google Drive content. Built with Tauri, Rust, and React. Privacy-first by design.

📂 Media Library

Browse movies and TV shows from local storage or Google Drive with rich metadata.

▶️ Multiple Players

Play via MPV (bundled), VLC, or the built-in player. Audio/subtitle track selection.

☁️ Cloud Streaming

Stream video directly from your Google Drive without downloading. ZIP archive support.

👥 Watch Together

Synchronized playback with friends using your own Cloudflare Worker relay.

🏷️ Metadata

Automatic metadata from TMDB, IMDb ratings, cast info, and episode details.

🔒 No Telemetry

Zero analytics, zero tracking, zero data collection. Verifiable open source.

Quick Start

  1. Download the latest release from GitHub Releases.
  2. Install and launch SlasshyVault.
  3. Sign in with Google to access your Drive.
  4. Optionally index your Drive folders in Settings.
  5. Add local media folders or point to Google Drive folders.
  6. Browse, search, and play your media.

Required: Windows 10+, Google account with Drive access. MPV is bundled with the installer.

Configuration

Settings are stored locally in media_config.json inside the app data directory. All fields can be configured through the Settings UI.

API Keys

SlasshyVault uses TMDB for metadata and imdbapi.dev (free) for IMDb ratings. You can optionally provide your own keys:

Watch Together

Watch Together synchronizes playback across users via a WebSocket relay. You can use a hosted relay or deploy your own.

Option 1: Deploy to Cloudflare (one-click)

  1. Go to Settings → Watch Together.
  2. Enter a Cloudflare API token with Workers Scripts: Edit permission.
  3. Click Deploy Relay. The app uploads the worker to your account automatically.
  4. The relay URL is saved to settings. Share it with friends to join rooms.

Option 2: Manual URL

Paste any wss:// relay URL into Settings → Watch Together. The relay runs the protocol defined in src-tauri/relay/worker.js.

Option 3: Deploy manually with wrangler

git clone https://github.com/SlasshyOverhere/SlasshyVault
cd SlasshyVault/src-tauri/relay
# Copy worker.js to a new directory
mkdir my-relay && cd my-relay
cp ../worker.js .
cat > wrangler.toml << 'EOF'
name = "slasshyvault-together-relay"
main = "worker.js"
compatibility_date = "2026-05-01"

[[durable_objects.bindings]]
name = "ROOM"
class_name = "Room"

[[migrations]]
tag = "v1"
new_sqlite_classes = ["Room"]
EOF
npx wrangler deploy

Note the wss:// URL and paste it in Settings.

Self-Hosting the Backend

The only external dependency is the OAuth backend for Google sign-in. It is intentionally minimal — a single Express file with no database.

Deploy on Render (free)

1. Fork the repo on GitHub
2. Go to dashboard.render.com → New Web Service
3. Connect your fork, settings:
   - Root directory: SlasshyVault-Backend
   - Build: npm install
   - Start: npm start
4. Set env vars:
   GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, REDIRECT_URI
5. Deploy
6. Add your redirect URI to Google Cloud Console:
   https://your-app.onrender.com/auth/callback

Run on a VPS

git clone https://github.com/SlasshyOverhere/SlasshyVault
cd SlasshyVault/SlasshyVault-Backend
npm install

# Create .env
cat > .env << 'EOF'
GOOGLE_CLIENT_ID=xxx
GOOGLE_CLIENT_SECRET=xxx
REDIRECT_URI=https://your-domain.com/auth/callback
PORT=3001
OAUTH_CALLBACK_URL=http://localhost:8085/callback
EOF

node server.js

Pointing the App to Your Backend

The auth server URL is hardcoded in src-tauri/src/gdrive.rs at line 18:

const AUTH_SERVER_URL: &str = "https://slasshyvault.onrender.com";

To use your own, set the environment variable STREAMVAULT_AUTH_SERVER_URL at build time, or change the constant and rebuild. This is intentional — it prevents phishing by ensuring the app only connects to a trusted auth server.

Endpoint reference The backend exposes only these routes: GET / · GET /health · GET /auth/google · GET /auth/callback · GET /auth/session/:id · POST /auth/refresh

Security & Privacy

SlasshyVault is designed so you can verify every claim by reading the source. Below is a summary of key properties with evidence.

Data Collection

None. The app has no analytics, no telemetry, no crash reporting, no tracking of any kind. There is no "phone home" on launch.

# Verify yourself:
grep -rni "analytics\|telemetry\|sentry\|datadog\|mixpanel\|amplitude" src/ src-tauri/
# → No results

OAuth Backend

The backend is a stateless Node.js server (~120 lines). It stores OAuth state in memory for a maximum of 5 minutes, then deletes it. There is no database, no Redis, no persistent storage, and no logging of user data.

Tokens

OAuth tokens are stored locally in the app data directory, obfuscated. They are used exclusively for Drive API calls and are never transmitted to any server other than Google.

Watch Together Relay

The relay runs on your own Cloudflare account. It keeps room state in Durable Object memory only — no database, no persistence. State is lost when the last participant leaves.

Third-Party API Calls

All metadata requests (TMDB, imdbapi, OMDb) are made directly from your machine. They include only media IDs or search queries — no personal information.

Open source auditability The full source is at github.com/SlasshyOverhere/SlasshyVault. You can inspect every network call, every data path, and every dependency.

FAQ

Does SlasshyVault collect any data?

No. No telemetry, analytics, crash reports, or usage tracking. The app makes zero network requests on launch. All data stays on your machine or in your Google Drive.

What does the OAuth backend do?

It exchanges Google's one-time authorization code for access and refresh tokens. It stores a session ID in memory for 5 minutes max, then deletes it. No database, no logs, no persistent storage.

Where is my data stored?

Watch history and metadata cache → local SQLite. Config and tokens → media_config.json. Media files → your local disk or Google Drive. Nothing is uploaded to any server.

Can I use my own TMDB key?

Yes. Go to Settings → API Keys and enter your TMDB API key or access token. This bypasses any shared pools and gives you dedicated rate limits.

How does Watch Together work?

One user creates a room. Others join by room code. Playback state (play/pause/seek) is relayed through a Cloudflare Worker running on your own account. The worker source is bundled in the app and is fully auditable.

Is there a mobile version?

No. SlasshyVault is a desktop app for Windows only. Built with Tauri, not a mobile framework.

The relay is a Cloudflare Worker deployed to your own account. It uses Durable Objects for ephemeral room state — no persistence, no database. State is lost when the last participant leaves or after 6 hours of inactivity.
